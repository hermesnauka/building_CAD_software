# Project Agents & Roles

This document outlines the key roles (and AI agent personas) responsible for executing the SSDLC for the WordPress platform.

## 1. Project Manager / Scrum Master
*   **Responsibilities:** Manages the project roadmap, ensures SSDLC milestones are met, and facilitates communication between security and development teams.

## 2. Security Architect (SSDLC Lead)
*   **Responsibilities:** Threat modeling, defining security requirements, ensuring PCI-DSS compliance for payment integrations (BLIK, PayPal, Bank logins), and reviewing architecture.

## 3. Lead WordPress Developer
*   **Responsibilities:** Implementing secure custom themes/plugins, configuring WooCommerce/e-commerce plugins securely, optimizing MySQL queries, and ensuring fast content publication capabilities.

## 4. QA & Security Tester
*   **Responsibilities:** Running Static Application Security Testing (SAST) and Dynamic Application Security Testing (DAST) tools, vulnerability scanning (e.g., WPScan), and verifying Role-Based Access Control (RBAC).

## 5. Content & E-Commerce Manager (Product Owner)
*   **Responsibilities:** Managing user stories, prioritizing educational content regarding construction tech, and overseeing the CAD software sales catalog.
